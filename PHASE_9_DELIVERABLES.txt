================================================================================
                              CACHING STRATEGY
================================================================================

CACHE FIRST (App Shell)
  Assets: HTML, CSS, JS, icons, Google Fonts
  Behavior: Serve from cache immediately, update in background
  Purpose: Fastest repeat loads (~100ms)
  TTL: No expiration (version-based updates)

NETWORK FIRST (APIs)
  Routes: /chat, /quiz, /estimate, /progress
  Behavior: Try network first, fall back to cached response
  Purpose: Fresh data with offline fallback
  TTL: No explicit TTL (browser decides)

NETWORK ONLY (Dev Routes)
  Routes: /dev/metrics, /admin, /cache-stats
  Behavior: Always go to server, never cache
  Purpose: Admin tools always need live server
  TTL: Not cached

NEVER CACHED (POST Requests)
  Method: POST (all routes)
  Behavior: Network only, never cache response
  Purpose: User messages must reach server
  Rationale: Prevents duplicate processing
